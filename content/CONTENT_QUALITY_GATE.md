# See The Next Move Content Quality Gate

Use this gate for every automated or manually drafted content item before publication.

The purpose is to make sure each piece is commercially useful, credible, problem-led and aligned to the See The Next Move methodology.

## Publication Standard

Every content item must:

- Start from a real Finance leadership problem
- Include a referenceable use case
- Explain the measurable value logic
- Connect to a recognised framework, emerging practice or best practice
- Help the reader solve a practical problem
- Reinforce the methodology: define the roadmap, land the change, govern delivery
- Lead naturally to the free assistant and/or paid diagnostic
- Avoid hype, generic AI commentary and unsupported statistics
- Use British English

## Scoring Checklist

Score each dimension from 1 to 5.

1 = weak or missing  
3 = acceptable but needs sharpening  
5 = strong and ready to publish

| Dimension | Score | Pass Standard |
| --- | ---: | --- |
| Real Finance leadership problem |  | The issue is specific, recognisable and senior enough to matter |
| Referenceable use case |  | The piece includes a practical Finance example, not abstract commentary |
| Measurable value logic |  | Benefits are linked to cycle time, quality, risk, capacity, adoption or decision impact |
| Framework or best practice |  | The advice is structured through a method, lens or recognised practice |
| Practical problem solving |  | The reader can take action after reading |
| Methodology alignment |  | Clear link to roadmap sequencing, adoption and programme oversight |
| Commercial pathway |  | The piece naturally points to the AI assistant and/or paid diagnostic |
| Credibility |  | Claims are proportionate, experience-based or clearly framed as judgement |
| Originality |  | The piece has a clear point of view rather than generic AI content |
| British English |  | Spelling and terminology are UK-aligned |

## Minimum Threshold

Do not publish if:

- Any score is below 4
- The piece is mainly educational but has no commercial pathway
- The piece describes AI capability without a Finance operating model implication
- The piece relies on unsupported statistics or vague market claims
- The paid diagnostic CTA feels disconnected from the problem

## Required Content Structure

Each full content pack should contain:

- Working title
- SEO slug
- Meta description
- Target search intent
- Core problem
- Referenceable use case
- Measurable achievement logic
- Framework or emerging practice
- Practical actions
- Change risk and mitigation
- Link to the See The Next Move methodology
- Free AI assistant prompt
- Paid diagnostic CTA
- Three personal LinkedIn posts
- One company LinkedIn post
- Visual concept
- Quality gate score

## Preferred CTA Language

Use direct, low-friction wording:

"Use the free AI Finance Playbook assistant to prepare your thinking. If you want an independent view of your roadmap, adoption risks and next actions, book a paid AI Finance Diagnostic."

## Avoid

- "AI will revolutionise Finance" without explaining how
- Generic listicles
- Vendor-led claims
- Unsupported benchmarks
- Fear-based selling
- Content that gives away vague advice but no clear next step
- A free consultation route

## Funnel Logic

Each content item should move the reader through this sequence:

Problem recognised -> practical use case understood -> value logic made visible -> methodology credibility established -> free assistant used for preparation -> paid diagnostic booked for expert judgement.

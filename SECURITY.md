# Security Policy

## Supported Scope

This repository hosts the public static website for See The Next Move.

Everything committed to this repository should be treated as public. Do not commit client information, credentials, private documents, payment account details, meeting notes, transcripts containing confidential material, or local OneDrive migration outputs.

## Reporting A Security Issue

If you find a security issue affecting this website or repository, contact:

gilles.bonelli@seethenextmove.com

Please include:

- The page, file, or behaviour affected
- A concise description of the issue
- Steps to reproduce, if applicable
- Any evidence that sensitive data may have been exposed

## Repository Handling Rules

- Keep GitHub two-factor authentication enabled.
- Keep repository access limited to trusted collaborators only.
- Use branch protection on `main`.
- Keep GitHub Pages HTTPS enforced.
- Do not store secrets in repository files, GitHub Pages content, screenshots, exported transcripts, or generated content packs.
- Review any new automation before granting write permissions.

## Public Site Data Rules

The website may contain public marketing content, public contact details, public booking links, public thought leadership, and public resources.

The website must not contain:

- Passwords, API keys, personal access tokens, OAuth secrets, or private keys
- Stripe, PayPal, Calendly, Google, Cloudflare, Namecheap, or GitHub credentials
- Client confidential information
- Private OneDrive migration logs
- Private CV drafts or financial documents
